# Linked Lists

A Linked List is a linear data structure where elements are connected using pointers instead of contiguous memory.

Unlike arrays, linked lists do not support direct access by index. To reach a particular node, we must traverse the list step by step. As a result, solving linked list problems requires careful pointer manipulation and a good understanding of node connections.

---

## Types of Linked Lists

### Singly Linked List

Each node contains data and a pointer to the next node.

```text
1 → 2 → 3 → 4 → NULL
```

### Doubly Linked List

Each node contains pointers to both the previous and next nodes.

```text
NULL ← 1 ⇄ 2 ⇄ 3 ⇄ 4 → NULL
```

### Circular Linked List

The last node points back to the first node.

```text
1 → 2 → 3 → 4
↑           ↓
└───────────┘
```

---

## Common Patterns

The following patterns form the foundation for solving most linked list problems:

1. Fast & Slow Pointers
2. Reversal
3. Merge Lists
4. Dummy Node Technique
5. Cycle Detection

These patterns appear repeatedly in coding interviews and competitive programming problems.

## Applications of Linked Lists

Linked Lists are commonly used in:

* Stacks and Queues
* Browser History Navigation
* Undo and Redo Operations
* Music and Image Playlists
* Hash Tables (Separate Chaining)
* LRU Cache
* Graph Adjacency Lists

---

## Important Note

* Most interview problems use **Singly Linked Lists**.
* Doubly Linked Lists are commonly used in **LRU Cache**, browser history, and undo/redo operations.
* Circular Linked Lists are mainly used in **cycle-related problems** and round-robin scheduling.
* Linked List problems are more about **pointer manipulation** than mathematical formulas.
* Drawing diagrams and visualizing pointer movement can greatly improve understanding and help avoid mistakes.

Mastering these core patterns will make it easier to solve more advanced linked list problems with confidence.
