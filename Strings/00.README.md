# Strings

Strings are one of the most important data structures in programming. A string is simply a sequence of characters, which means many techniques used for arrays can also be applied to strings.

Because of this, several patterns such as Two Pointers, Sliding Window, Hashing, and Prefix Sum do not need to be learned again from scratch. The same ideas work almost identically on strings.

---

# Relationship Between Arrays and Strings

You can think of a string as an array of characters.

For example:

```
String: "hello"

Index:  0 1 2 3 4
Chars:  h e l l o
```

Since strings have indices, we can traverse them, use pointers, maintain windows, and perform prefix calculations just like arrays.

# Recommended Prerequisites

Before studying string-specific algorithms, it is recommended to understand:

1. Arrays → Two Pointers
2. Arrays → Hashing
3. Arrays → Prefix Sum
4. Arrays → Sliding Window

These patterns form the foundation for solving many string problems.

---

# String-Specific Patterns

Although many techniques are shared with arrays, some algorithms are unique to strings.

## 1. String Simulation

The simplest type of string problems.

### Examples

* Reverse String
* String Compression
* Roman to Integer
* Valid Palindrome

These problems mainly involve following rules carefully and manipulating characters.

---

## 2. KMP Algorithm

KMP (Knuth-Morris-Pratt) is an efficient pattern matching algorithm.

### Used For

* Finding a pattern inside a string
* Avoiding unnecessary comparisons
* Linear-time substring search

### Examples

* Find the Index of the First Occurrence in a String
* Repeated Substring Pattern

---

## 3. Rabin-Karp Algorithm

Rabin-Karp uses rolling hashes to compare substrings efficiently.

### Used For

* Pattern matching
* Multiple substring searches
* Duplicate substring problems

### Examples

* Repeated DNA Sequences
* Pattern Searching

---

## 4. Trie

A Trie is a tree-like structure designed specifically for strings.

### Used For

* Prefix search
* Dictionary problems
* Autocomplete systems
* Word search problems

### Examples

* Implement Trie
* Design Add and Search Words
* Word Search II

---

# Learning Order

The recommended order for studying strings is:

```
String Simulation
        ↓
KMP Algorithm
        ↓
Rabin-Karp
        ↓
Trie
```

---

# Important Note

Strings are not completely separate from arrays.

Most string problems are solved by combining array patterns with string-specific algorithms.

Therefore, mastering the Arrays section first will make learning strings much easier.
